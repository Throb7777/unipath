# UniPATH Android

> 查看英文版：[README.md](./README.md)

> 这是 UniPATH 的 Android 应用。它负责接收分享链接，把任务提交给本地转发服务，并把任务状态再展示给用户。

## 🌟 这个应用能做什么

应用对应的流程是：

1. 你从其他 Android 应用分享一个链接
2. 本应用接收分享文本
3. 应用把任务发给本地转发服务
4. 转发服务调用当前处理方式
5. 应用显示 **转发状态**、**近期任务** 和最终 **结果摘要**

## ✅ 主要页面

这些名称与界面保持一致：

- `首页`
- `设置`
- `转发状态`
- `完整流程`

常用按钮：

- `测试连接`
- `保存`
- `中断任务`
- `复制结果`
- `查看完整流程`

## 📦 运行前准备

### 必需

- Android Studio
- Android SDK
- 一个正在运行的转发服务

### 建议准备

- Android 模拟器
- 本地 OpenClaw 环境，如果你要跑真实处理

## 🚀 构建并运行，按步骤来

### 第 1 步：打开 Android 项目

用 Android Studio 打开 `Android/` 目录。

### 第 2 步：等待 Gradle 同步完成

等待直到：

- Gradle 同步完成
- SDK 组件准备好

### 第 3 步：检查 `local.properties`

Android Studio 一般会自动生成这个文件。

它应该指向你本机的 Android SDK。

注意：

- `local.properties` 只保留在本地
- 不要把它提交到版本控制

### 第 4 步：运行应用

你可以选择：

- Android 模拟器
- Android 真机

然后点击 Android Studio 的 **Run**。

## 🔧 第一次打开应用后的设置步骤

请按下面顺序操作。

### 第 1 步：打开 `设置`

在应用里点击 **设置**。

### 第 2 步：填写转发服务地址

填写转发服务的基础地址。

常见写法：

- Android 模拟器连接 Windows 主机：
  - `http://10.0.2.2:8080`
- Android 真机连接局域网电脑：
  - `http://<你的局域网 IP>:8080`

### 第 3 步：点击 `测试连接`

这一步会检查：

- 转发服务是否能访问
- 处理模式列表是否能拉取成功

### 第 4 步：选择一个 `处理模式`

连接测试成功后，选择一个处理模式。

### 第 5 步：点击 `保存`

这样应用会把设置保存到本地。

## 📱 如何跑一次真实测试

### 第 1 步：打开一条支持的文章链接

例如一篇微信公众号文章链接。

### 第 2 步：把链接分享给本应用

使用 Android 系统分享，然后选择本应用。

### 第 3 步：确认提交

应用会显示：

- 来源
- 规范化后的链接
- 当前处理模式

然后点击 **发送**。

### 第 4 步：查看 `转发状态`

你可以在这里查看：

- 当前状态
- 结果摘要
- 处理步骤
- 后续建议

### 第 5 步：需要时打开 `完整流程`

如果你想看完整的步骤记录，就点击 **查看完整流程**。

## 🧭 各页面分别适合做什么

### `首页`

适合：

- 查看当前转发服务设置
- 查看**近期任务**
- 快速重新打开某条任务

### `设置`

适合：

- 修改转发服务地址
- 点击 `测试连接`
- 选择处理模式
- 切换应用语言
- 点击 `保存`

### `转发状态`

适合：

- 查看当前状态
- 查看结果摘要
- 查看后续建议
- 复制诊断摘要
- 用 `中断任务` 停掉一条任务

### `完整流程`

适合：

- 查看完整处理步骤
- 判断任务停在了哪一步

## 🧪 构建产物

如果你想在本地生成 debug APK，可以用 Android Studio，或者执行：

```bash
./gradlew assembleDebug
```

Windows：

```powershell
.\gradlew.bat assembleDebug
```

## ⚠️ 重要说明

- 这个应用必须依赖正在运行的转发服务。
- 应用本身不会独立完成文章处理。
- 真正的文章处理结果取决于 Relay 中选择的处理方式。
